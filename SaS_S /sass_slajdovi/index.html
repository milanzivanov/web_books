CSS predprocesori

  • Eng. CSS preprocessors
  • Uvode nove mogućnosti proširivanjem CSS jezika.
  • Proširenja se uglavnom baziraju na nedostacima samog CSS-a i na dobrim praksama i mehanizmima iz programskih jezika i objektno-orijentisanog programiranja.
  • Kreirani sa ciljem da olakšaju kreiranje CSS sadržaja kako bi se održala organizovanost i lakoća održavanja koda bez obzira na veličinu projekta.

CSS predprocesori

  • Predstavnici: LESS, Sass, Stylus, Switch CSS, …
  • Svaki od njih uglavnom omogućava sličan skup mogućnosti, ali sa različitim sintaksama.
  • S obzirom da browseri direktno ne podržavaju ni jedan od CSS predprocesora, kod napisan po pravilima predprocesora se prevodi u čisti CSS kod.
  • Uglavnom su svi unazad kompatibilni sa čistim CSS-om.
  • U većini slučajeva se izbor svodi na LESS ili Sass.

LESS.js

  • lesscss.org
  • Baziran na JavaScript jeziku i Node.js biblioteci.
  • Instalira se tako što se less.js datoteka linkuje u željeni HTML dokument (može i kao deo Node paketa) putem script taga.
  • LESS kod se piše u datoteke sa ekstenzijom .less koje se takođe linkuju u HTML dokument pomoću link taga.
  • LESS kod se kompajlira u CSS dinamički prilikom učitavanja stranice.

SASS

  • sass-lang.com
  • Syntactically Awesome Stylesheets
  • Bazira se na Ruby programskom jeziku.
  • Instalira se putem RubyGems platforme.
  • Može se koristiti iz komandne linije ili integrisan u postojeće grafičko okruženje za razvoj HTML stranica (najčešći izbor su Codekit ili Komass) ili tekstalni editor.
  • Sass kod se piše u datotekama sa ekstenzijom .scss ili .sasss koje se kompajliraju u CSS na neki od spomenutih načina.

SASS

Nakon instaliranja Ruby kompajlera, sass se instalira naredbom:

	    	    		gem install sass

Kompajliranje Sass datoteka iz komandne linije

Ručno:

sass ulaz.scss izlaz.css

Automatski:

sass --watch ulaz.scss:izlaz.css

Može i nad celim direktorijumima:

sass --watch sass:css

sass --watch komanda

  • Automatski prevodi navedenu .scss datoteku u CSS prilikom svakog snimanja.
  • Ulazna i izlazna datoteka se navode razdvojene dvotačkom.
  • Ukoliko se ne navede izlazna CSS datoteka, ona će biti automatski kreirana sa istim imenom kao i u istom direktorijumu i ulazna Sass datoteka.
  • Sass i CSS datoteke ne moraju da budu istim direktorijumima.
  • Osim pojedinačnih datoteka, komanda može automatski da kompajlira i cele direktorijume.

SASS sintakse: Sass

  • Prvobitna Sass sintaksa.
  • Kreirana po uzoru na Haml jezik.
  • Ne koristi znakove {, } i ; za razdvajanje blokova i pravila, već za to koristi indentaciju.
  • Zbog ovih razlika u odnosu na čisti CSS, potrebno je neko vreme za učenje i privikavanje na novu sintaksu prilikom prelaska na Sass.
  • Kod se piše u datotekama sa ekstenzjom .sass.

SASS sintakse: SCSS

  • Sassy CSS
  • Vrlo slična CSS jeziku, koristi se od Sass vezije 3.
  • Može se smatrati nadogradnjom CSS jezika.
  • Kod se piše u datotekama sa .scss ekstenzijom.
  • Koristićemo je na ovom kursu.

Sass sintakse: Konverzija

  • Ukoliko je potrebno, Sass omogućava konverziju između svojih sintaksi putem sass-convert komande.
  • Sass –> SCSS

  • SCSS –> Sass 
    sass-convert style.scss style.sass

Sass promenljive

  • Jedan od osnovnih (i najkorisnijih) koncepata koji predprocesori uvode u CSS je mogućnost smeštanja vrednosti u promenljive.
  • Omogućavaju smeštanje često korištenih CSS vrednosti po uzoru na programske jezike.
  • Vrlo korisne za definisanje boja ili određenih numeričkih vrednosti koje se koriste na više mesta.
  • Imena promenljivih u Sass jeziku počinju sa znakom $ i ne smeju sadržavati razmake.
  • Vrednosti promenljivih se navode razdvojene dvotačkom od imena.

Sass promenljive

						
$boja-pozadine: #000;
$boja-teksta: #fff;

* {
	background: $boja-pozadine;
	color: $boja-teksta; 
}
  • Prilikom kompajliranja, svaka pojava imena promenljive će biti zamenjena njenom vrednošću.

Ugnježđeni selektori

  • CSS predprocesori omogućavaju pisanje CSS selektora i njihovih pripadajućih pravila unutar drugih selektora.
  • Ugnježdavanje selektora čini CSS kod lakšim za pisanje jer se izbegavaju suvišna navođenja roditeljskih elemenata u selektorima za svaki pripadajući element.
  • CSS postaje i lakši za čitanje jer ugnježđena struktura podseća na hijerarhiju HTML dokumenta.

Ugnježđeni selektori

Umesto da pišemo:

						
nav ul { 
	/* Pravila za glavnu listu */ 
}
nav ul li { 
	/* Pravila za stavke liste */ 
}
nav ul li a { 
	/* Pravila za linkove u stavkama */ 
}

Ugnježđeni selektori

Možemo pisati:

						
nav  ul {
	/* Pravila za glavnu listu */
	li {
		/*  Pravila za stavke liste */
		a {
			/* Pravila za linkove u stavkama */
		}
	}
}

Ugnježđeni selektori

ZADATAK:

Primer z1-nesting_selectors izmeniti tako da se koriste Sass ugnježđeni selektori i promenljive. Primer prevesti u CSS i prikazati rezultat u browseru.

Ugnježđeni selektori

  • Obratiti pažnju da mi nemamo kontrolu nad generisanim CSS kodom, tako da nekada upotreba ugnježđenih selektora može rezultovati CSS selektorima vrlo velike specifičnosti, što može uticati na njigov opseg i otežati izmene određenih pravila.
  • Međutim, uz malo planiranja i sticanjem više iskustva, ovakve situacije se lako mogu izbeći.

Ugnježđeni selektori

  • Drugi problem koji se javlja prilikom prevelike upotrebe ugnježđenih selektora je povećanje zavisnosti CSS stilova od DOM strukure HTML dokumenta.
  • Prilikom promene stukture HTML dokumenta, potrebno je menjati Sass kod.
  • Kako bi se izbegao ovaj problem, neki autori smatraju da Sass kod ne bi trebalo ugnježdavati preko 4. nivoa.

Referenca na roditeljski selektor

Znak &

						
a {
	color: #f00;
	
	&:hover {
		color: #00f;
	}
}

Svaka pojava znaka & biće zamenjena selekrom koji je direktan roditelj pravila u kojem je znak naveden (u ovom slučaju a).

Nasleđivanje selektora

  • Implementirano po uzoru na nasleđivanje iz OOP.
  • Još jedna mera smanjenja redundantnosti CSS koda.
  • Selektor koji nasleđuje drugi selektor nasleđuje sva pravila iz slelektora koji nasleđuje.
  • Pored nasleđenih pravila, selektor može da uvodi svoja specifična pravila.

Nasleđivanje selektora

  • Deklariše se anotacijom @extend nakon koje se navodi selektor koji se nasleđuje.
						
.red-link {
	@extend .link;
	color: #f00;
}
  • Pod uslovom da klasa .link postoji, klasa .red-link će naslediti sva njena CSS pravila i dodati svoje pravilo za boju teksta.
  • PRIMER: 4. extend

Nasleđivanje selektora

  • Selektor koji se nasleđuje mora postojati, inače Sass kompajler javlja grešku.
  • Nasleđena pravila neće biti samo ubačena na mesto @extend direktive, već će biti izgenerisano zajedničko pravilo za nasleđeni selekor i selektor koji nasleđuje.
  • Nasleđivanje ne radi u kombinaciji sa media-query mehanizmom.

Mixini

  • Omogućavaju definisanje imenovanih blokova CSS pravila koji se mogu koristiti na više mesta.
  • Definišu se direktivom @mixin
    • 
@mixin moj-prvi-mixin {
	...
}
  • Kreirani mixin se uključuje u neko pravilo direktivom @include nakon koje se navodi ime mixina.
  • U generisanom CSS kodu će na mestu svake od @include direktiva biti ubačen kod iz mixina sa tim imenom.

Mixini

  • Mixini mogu da sadrže kompletne CSS blokove uključujući selektore i pripadajuća pravila ili samo skup pravila.
  • Ukoliko mixin predstavlja skup CSS blokova, on može biti uključen izvan CSS bloka.
  • Mixini mogu da uključuju druge mixine.
  • PRIMER: 5. mixins

Mixini

  • Mixini mogu da imaju proizvoljan broj ulaznih parametara koji su kao lokalne promenljive dostupni u telu mixina.
  • Parametri se navode nakon imena mixina u zagradi, razdvojeni zarezom.
  • Parametri mogu imati i podrazumevane (default) vrednosti.
    • 				    		
@mixin moj-mixin($param1: vrednost1, $param2: vrednost2) {
	…
}

PRIMER: 6. mixins_arguments

Mixini

  • Mixini predstavljaju vrlo feksibilno i elegentno rešenje za kreiranje ponovo upotrebljivog koda (DRY pravilo).
  • Zbog toga što predstavljaju potpunije rešenje od nasljeđivanja selektora, mnogi autori predlažu korištenje mixina potpuno umesto nasleđivanja.
  • Vrlo su korisni za vendor-prefixed pravila.
  • Danas se na internetu može naći veliki broj gotovih mixina koji rešavaju mnoge praktične CSS probleme.

Numerički operatori

  • Sass omogućava upotrebu osnovnih matematičkih operatora u svojim pravilima.
  • Podržane operacije: +, -, *, /, %
  • Operacije se mogu vršiti nad konkretnim vrednostima I nad promenljivama.
  • U kombinaciji sa promeljivama, operatori predstavljaju vrlo korisno proširenje CSS-a.

Numerički operatori

Ograničenja:

  • Prioriteti operatora su isti kao u osnovnoj aritmetici.
  • U izrazima je moguće koristiti samo komaptibilne jedinice (px i in su kompatibilni, ali px i em nisu).
  • Množenjem dva broja istih jedinica dobija se rezultat u kvadratnim jedinicama (10px * 10px = 100px * px), pa se u ovim slučajevima preporučuje izostavljanje jedinice kod drugog operanda.

PRIMER: 7. operations_grid

Numerički operatori: Delenje

Problem:

  • Znak / se u CSS koristi za kraći zapis nekih pravila (pravilo font: 12px / 18px; postavlja font-size na 12px i line-height na 18px).
  • Isti znak se u Sass koristi za delenje
  • S obzirom da je Sass proširenje CSS sintakse, CSS interpretacija mora biti ispoštovana.

Numerički operatori: Delenje

Znak / će biti iskorišćen za delenje u sledećim slučajevima:

  • Ako su operandi smešteni u promeljive ili predstavljaju povratne vrednosti funkcija,
  • Ako izraz zaokružimo običnim zagradama,
  • Ako se izraz koristi kao deo nekog drugog aritmetičkog izraza.

U svim ostalim slučajevima će izrazi koji sadrže / biti direktno preneseni u CSS.

Numerički operatori: Delenje

Izraz Interpretacija
font: 10px/8px; CSS izraz, bez delenja
width: $width/2; Izraz sadrži promenljivu, delenje
width: round(1.5) / 2; Izraz sadrži funkciju, delenje
height: (500px/2); Izraz je u zagradama, delenje
margin: 5px + 8px/2px; Deo drugog aritmetičkog izraza, delenje

Numerički operatori: Oduzimanje

  • Slično ako /, i operator - može imati različita značenja u CSS-u i Sass-u (operator oduzimanja, negativan predznak, unarni operator negacije).
  • Kako bi se izbegla pogrešna tumačenja, preporučuje se poštovanje sledećih smernica:
    • Prilikom pisanja izraza oduzimanja, staviti razmak ispred i iza znaka -($a - $b).
    • Prilikom pisanja negativnih brojeva ili unarne negacije, staviti razmak ispred ali ne i iza znaka -( -54).

Numerički operatori: Boje

  • Sass omogućava izvođenje aritmetičkih operacija nad bojama.
  • Operator se primenjuje na svaku komponentu posebno (crvena, zelena i plava).
  • U operacijama je moguće kombinovati boje (podržani su svi validni CSS zapisi) I brojeve.
  • Međutim, za manipulaciju nad bojama, preporučuje se upotreba ugrađenih Sass funcija.

Operatori jednakosti

  • Proveravaju da li su dve vrednosti jednake.
  • Vraćaju logičku vrednost true ili false.
  • Podržani operatori:
    • == - Vraća true ako su operandi jednaki
    • != - Vraća true ako operandi nisu jednaki
  • Koriste se u logičkim izrazima.

Operatori poređenja i logički operatori

  • Sass podržava standardne operatore poređenja i osnovne logičke operatore.
  • Podržani operatori poređenja:
    • >, <, >=, <=
  • Podržani logicki operatori:
    • and, or, not

Kontrola toka

  • Sass (kao i svi ostali CSS predprocesori) uvodi mogućnost kontrole toka izvršavanja koda putem kontrolnih struktura i petlji.
  • Podržana je kontrolna struktura if-else.
  • Od petlji postoje for, each i while.
  • Ove strukture predstavljaju vrlo korisno proširenje, pogotovo u kombinaciji sa predstavljenim mehanizmima.

if-else

  • Kreirana po uzoru na if-else direktive iz programskih jezika.
  • Služi za primenu CSS stila u zavisnosti od logičkog uslova.
  • Može da ima proizvoljan broj grananja.
  • Izraz u uslovu se smatra tačnim ukoliko vraća vrednost koja nije false ili null.

if-else

						
$size: small;
p {
	@if $size == small { font-size: 10px; }
	@else if $size == large { font-size: 16px; }
	@else if $size == xlarge { font-size: 26px; }
	@else { font-size: 12px; }
}

PRIMER: 8. if-else

for petlja

Služi za iterativno generisanje CSS pravila.

						
@for $var from <start> through <end> { 
	/* . . . */  
}
@for $var from <start> to <end> { 
	/* . . . */
}

$var – ime lokalne promenljive koja se menja u svakoj iteraciji

start i end – bilo koji Sass izraz koji vraća celobrojnu vrednost.

Varijanta sa through uzima u obzir i end vrednost dok varijanta sa to ne uzima.

for petlja

						
@for $i from 1 trough 3 {
    .item-#{$i} { width: 2em * $i; }
}

Rezultat:

						
.item-1 { width: 2em; }
.item-2 { width: 4em; }
.item-3 { width: 6em; }
